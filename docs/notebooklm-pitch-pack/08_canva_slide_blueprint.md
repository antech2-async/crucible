# Canva Slide Blueprint

This is the recommended final pitch deck structure.

Format: 12 slides  
Pitch duration: 5 minutes  
Language: English  
Audience: 0G APAC Hackathon judges and general audience  
Style: dark command-center infrastructure deck with amber and cyan accents

## Slide 1 - Cover

Title: Crucible  
Headline: Programmable accountability for autonomous AI agents  
On-slide copy:

- Built on 0G
- Agentic Infrastructure + Agentic Economy
- 0G APAC Hackathon

Visual direction:

- Full-screen dark Arena-inspired background
- Abstract agent network or dashboard mesh
- Crucible wordmark large
- Small 0G badge

Speaker note:

"Crucible is an accountability layer for autonomous AI agents. It lets agents collaborate with stake, verification, history, and consequences."

## Slide 2 - Problem

Title: The Missing Trust Layer  
Headline: Agents can work. They still cannot trust each other.  
On-slide copy:

- Agent frameworks can execute tasks.
- Multi-agent collaboration is growing.
- But bad agents can submit weak work, abandon identity, and repeat.
- Reputation alone is not enforcement.

Visual direction:

- Two or three agent cards connected by broken lines
- One card red: "Defected"
- Payment stuck in the middle

Speaker note:

"The problem is not whether agents can generate output. The problem is what happens when independent agents from different owners need to trust each other."

## Slide 3 - Why Now

Title: Why Now  
Headline: The agent economy is arriving before its accountability layer.  
On-slide copy:

- Open-source agent frameworks are becoming practical.
- Agents increasingly use tools, APIs, wallets, and workflows.
- Marketplaces need more than discovery and payment.
- They need verification, trust, and consequences.

Visual direction:

- Timeline or market wave
- Agent frameworks on the left
- "Accountability gap" in the middle
- Crucible on the right

Speaker note:

"Frameworks like OpenClaw, LangChain, AutoGen, and CrewAI help agents operate. Crucible answers the next question: how do strangers' agents safely collaborate?"

## Slide 4 - Solution

Title: The Solution  
Headline: Crucible turns agent behavior into economic terms.  
On-slide copy:

- Agents register with identity and capabilities.
- Owners deposit collateral.
- Tasks define objective criteria upfront.
- Smart contracts pay honest agents or slash failed ones.
- Behavior updates future trust and stake requirements.

Visual direction:

- Central Crucible box
- Inputs: identity, stake, criteria, history
- Outputs: payment, slash, trust update

Speaker note:

"Crucible does not just display reputation. It makes behavior affect future economics. Good agents get lower friction. Bad actors face higher collateral."

## Slide 5 - Product Loop

Title: Product Loop  
Headline: Register. Stake. Execute. Verify. Pay or slash.  
On-slide copy:

1. Register agent
2. Lock stake
3. Post task
4. Assign by capability and trust
5. Verify output
6. Update trust tier

Visual direction:

- Circular loop diagram
- Use icons for ID, vault, task, agent, shield, chart

Speaker note:

"Every task becomes a feedback loop. The result of today's work becomes tomorrow's collaboration terms."

## Slide 6 - Trust Economics

Title: Trust Economics  
Headline: Honest agents earn better terms. Bad actors get priced out.  
On-slide copy:

| Tier | Stake |
| --- | --- |
| Elite | 0.5x |
| High Trust | 0.75x |
| Moderate | 1.0x |
| Low Trust | 1.5x |
| New or Risky | 2.5x |

Microcopy:

Trust score = recent behavior + lifetime behavior - slash penalty

Visual direction:

- Vertical trust ladder
- Green or cyan at top, amber middle, red lower tier
- Small "BadBot after slash" marker moving down

Speaker note:

"The trust model is intentionally explainable. Recent behavior matters most, lifetime behavior stabilizes the score, and slash events create penalties."

## Slide 7 - Why 0G

Title: Why 0G  
Headline: Crucible needs the full AI-native stack.  
On-slide copy:

- 0G Chain: escrow, slashing, trust updates
- 0G Storage: agent history, task criteria, outputs
- 0G Compute: stronger native-agent verification
- INFT: persistent agent identity

Visual direction:

- 4-layer stack diagram
- Each layer connected to one Crucible function

Speaker note:

"This is not just using 0G as a chain. The mechanism needs storage, compute verification, identity, and enforcement working together."

## Slide 8 - Product Surface

Title: The Arena  
Headline: A live command center for agent accountability.  
On-slide copy:

- Agent roster and trust scores
- Task escrow and criteria
- Vault and stake readiness
- Live events and slashing signals
- Native and external agent classes

Visual direction:

- Use dashboard screenshot or mockup
- Highlight 3 areas: agent cards, task escrow, event feed

Speaker note:

"The Arena is how judges see the mechanism. It shows the trust network, active tasks, escrow state, and slashing events in one place."

## Slide 9 - Demo Story

Title: Demo  
Headline: Watch a strategic defector become expensive.  
On-slide copy:

1. Alice and BadBot are assigned.
2. Alice submits valid work.
3. BadBot fails criteria.
4. BadBot is slashed.
5. BadBot's next stake requirement rises.

Visual direction:

- 3-panel storyboard:
  - Before: BadBot looks reliable
  - During: BadBot fails criteria
  - After: slash and trust drop

Speaker note:

"The strongest demo is not an obviously bad agent. It is a strategic bad actor that looks reliable until it defects. Crucible makes that decision costly."

## Slide 10 - Technical Proof

Title: Technical Proof  
Headline: Built as a protocol, not just a mockup.  
On-slide copy:

- Solidity contracts: registry, vault, escrow, judge, trust calculator, INFT
- TypeScript assignment and verification engine
- 0G Storage integration path
- 0G Compute verification path for native agents
- Next.js Arena dashboard
- External-agent onboarding scripts

Visual direction:

- Architecture diagram with 3 columns: contracts, engine, frontend
- Small check marks for implemented surfaces

Speaker note:

"The codebase includes the protocol contracts, the off-chain coordinator, frontend Arena, and external-agent integration path. The demo is a visual layer over the mechanism."

Important note:

Do not invent mainnet addresses. If final verified mainnet links are not supplied, use "0G deployment proof" or "0G Galileo testnet deployment" depending on the final submission state.

## Slide 11 - Positioning

Title: Market Position  
Headline: We are not building another agent. We are building the accountability rail.  
On-slide copy:

- Agent frameworks run agents.
- Marketplaces discover agents.
- Payment rails move value.
- Crucible prices trust and enforces consequences.

Visual direction:

- Layer cake:
  - Agent apps
  - Agent marketplaces
  - Crucible accountability layer
  - 0G infrastructure

Speaker note:

"Crucible is complementary to the agent ecosystem. As agents become economic actors, they will need rails for trust, verification, and consequences."

## Slide 12 - Closing

Title: Closing  
Headline: The arena is the demo. The mechanism is the product.  
On-slide copy:

- Trust becomes measurable.
- Behavior changes terms.
- Slashing creates consequences.
- 0G makes the accountability layer verifiable.

Final line:

> Crucible makes autonomous agent collaboration economically accountable.

Visual direction:

- Large final statement
- Subtle agent network background
- Optional loop diagram watermark

Speaker note:

"Our vision is simple: if autonomous agents are going to collaborate and transact, trust cannot remain a static review score. It must become programmable infrastructure."

## Canva Design System

### Color

- Background: near-black or deep charcoal
- Primary accent: amber or warm gold
- Secondary accent: cyan or teal
- Success: green
- Failure/slash: red
- Text: white and soft gray

### Typography

- Title: bold, geometric sans
- Body: clean sans
- Technical labels: mono font

### Layout Rules

- One main idea per slide
- Use diagrams instead of paragraphs
- Keep body copy under 45 words per slide
- Put detailed reasoning in speaker notes
- Use product visuals for slides 8 and 9
- Use simplified architecture for slide 10
