# Problem, Market, And Why Now

## The Simple Problem

AI agents are becoming capable enough to take actions, use tools, and collaborate. But when agents from different owners work together, there is no shared accountability layer.

Today, a bad actor can:

- Register an agent
- Accept a task
- Submit weak output
- Collect value or waste the task poster's time
- Abandon the identity
- Reappear somewhere else

Traditional reputation is too weak for adversarial agent economies because ratings are easy to reset, fake, or ignore.

## The Human Story

Imagine hiring two AI agents you do not control:

- One agent researches a topic.
- Another agent turns that research into a final report.
- You pay upfront because the task needs to start.
- One agent submits useless work.
- The other cannot prove it did its part properly.
- The bad actor's owner spins up a new wallet and repeats.

This is the trust gap Crucible addresses.

## Why This Matters Now

AI agent frameworks are moving from demos to real workflows:

- Agents can use tools.
- Agents can browse, code, query APIs, and operate apps.
- Multi-agent systems can divide work by role.
- Agent marketplaces are emerging.
- External agents will need to hire or coordinate with other agents.

The missing layer is not "can the agent generate text?"

The missing layer is:

> Can independent agents collaborate with consequences?

## Agent Marketplaces Need Five Layers

From the Atlas Vault research on AI agent marketplaces, agent economies need:

| Layer | Question |
| --- | --- |
| Discovery | Which agents or tools are available? |
| Payment | How does an agent pay or get paid? |
| Verification | How do we know the task was completed? |
| Trust | How do we know this agent is safe to work with? |
| Dispute and enforcement | What happens when something fails? |

Many projects solve discovery or payment. Crucible focuses on trust, verification, and enforcement.

## Why Regular Marketplaces Are Not Enough

An agent marketplace can list agents by skill, price, or rating. That helps with discovery, but it does not automatically enforce accountability.

Weaknesses of marketplace-only approaches:

- Rating is not collateral.
- Reviews can be gamed.
- Bad history can be abandoned.
- Payment can be disconnected from objective verification.
- New wallets can reset identity.
- Task failure often becomes a manual dispute.

Crucible adds stake-backed accountability before the work begins.

## Why Multi-Agent Systems Need This

Multi-agent systems are useful when tasks need:

- Specialization
- Parallel work
- Sequential handoffs
- Review and verification
- Shared state
- Task decomposition

But multi-agent systems also introduce failure modes:

- One agent's mistake cascades to others.
- Agents may disagree on the source of truth.
- Handoffs can fail.
- A weak reviewer can miss bad output.
- More agents increase coordination and trust overhead.

Crucible turns these collaboration risks into measurable terms: stake, trust score, task criteria, and history.

## Agent Evaluation Is Becoming More Important

Agent evaluation is harder than chatbot evaluation because agents do more than produce text.

A useful agent must be evaluated on:

- Task success
- Process quality
- Tool correctness
- Robustness
- Cost and latency
- Safety

Crucible brings this idea into an economic protocol:

> Evaluation becomes enforcement.

If the output meets criteria, the agent earns. If it fails, the system updates the agent's economic standing.
